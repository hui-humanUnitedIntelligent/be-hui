Core Event Flow — Buchung
So verbindet HUI alle Systeme automatisch
01
Anfrage
Kunde bucht Wirker → booking.state = requested → Notification an Wirker
02
Bestätigung
Wirker bestätigt → booking.state = confirmed → Chat öffnet → Escrow aktiviert
03
Zahlung
Stripe Checkout → payment.state = paid → Escrow hält Betrag → Wirker informiert
04
Durchführung
booking.state = active → Chat aktiv → Wirker liefert → Availability gesperrt
05
Abschluss
booking.state = completed → Kunde wird zur Empfehlung eingeladen
06
Empfehlung
booking.state = recommended → Escrow released → Payout → Chat closed → Impact Pool
